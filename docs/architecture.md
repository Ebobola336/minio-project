# Архитектура проекта MinIO Object Storage

## Обзор

Проект реализует S3-совместимое объектное хранилище в режиме **Erasure Code** кластера из 4 нод MinIO. Все компоненты работают в изолированных Docker-контейнерах.

## Сетевая топология

```
Клиент (браузер / mc / SDK)
         |
         | HTTPS :9000 / :9443
         ▼
  ┌─────────────────┐
  │  Nginx Proxy    │  (frontend-net)
  │  TLS-терминация │
  └────────┬────────┘
           │  HTTP (backend-net)
     ┌─────┴──────┬──────────┬──────────┐
     ▼            ▼          ▼          ▼
  minio1       minio2     minio3     minio4
  (9000/9001)  (9000/9001) (9000/9001) (9000/9001)
     │            │          │          │
     └────────────┴──────────┴──────────┘
                 Erasure Code Storage
              (переживает потерю 2 из 4 нод)
                     │
              (monitoring-net)
                     │
              Prometheus → Grafana
```

## Сети Docker

| Сеть | Подсеть | Участники |
|---|---|---|
| frontend-net | 172.20.0.0/24 | nginx-proxy, prometheus, grafana |
| backend-net | 172.20.1.0/24 | nginx-proxy, minio1-4 |
| monitoring-net | 172.20.2.0/24 | minio1-4, prometheus, grafana |

## Erasure Code

MinIO использует алгоритм Reed-Solomon Erasure Coding:
- 4 ноды × 2 диска = 8 дисков итого
- Сохраняется до потери 4 дисков (n/2)
- Данные автоматически восстанавливаются при замене ноды

## Безопасность

- TLS 1.2/1.3 на всём внешнем трафике (Nginx)
- Docker Secrets для паролей (не попадают в логи)
- Непривилегированный запуск процессов
- Изолированные сети (дефолтная сеть не используется)
- Параметры hardening ядра через sysctl.d/
